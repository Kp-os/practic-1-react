# Звіт з Лабораторної роботи №1

## Фрагмент коду (Компонент Card)
\`\`\`jsx
import styles from './Card.module.css';

const Card = ({ children }) => {
return <div className={styles.card}>{children}</div>;
};

export default Card;
\`\`\`

## Відповіді на контрольні запитання

1. **Що таке Atomic Design і навіщо ми розділяємо компоненти на атоми та молекули?**
   Atomic Design — це підхід до архітектури, який передбачає поділ інтерфейсу на дрібні незалежні частини (атоми, молекули, організми). Це робить компоненти незалежними, спрощує їх підтримку та дозволяє легко перевикористовувати код у різних частинах додатка.

2. **Як працюють CSS Modules і як вони вирішують проблему глобальних імен класів?**
   CSS Modules локалізують стилі для конкретного компонента. Під час збірки проєкту вони генерують унікальні хеші для імен класів (наприклад, `.button_xyz123`), що повністю виключає конфлікти стилів між різними компонентами.

3. **Що таке props.children і в якому компоненті цієї лабораторної роботи ми його використали?**
   `props.children` — це спеціальний проп, який дозволяє передавати вкладені елементи або текст всередину компонента-обгортки. У цій роботі ми використали його в компоненті `Card`, щоб він міг працювати як універсальний контейнер для будь-якого контенту.

4. **Чому атрибут HTML class у JSX записується як className?**
   Оскільки JSX компілюється у звичайний JavaScript, а слово `class` є зарезервованим ключовим словом у JavaScript (використовується для створення класів ООП), у React використовується `className` для позначення CSS-класів.
