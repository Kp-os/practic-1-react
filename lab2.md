## [cite_start]Фрагмент коду з App.jsx (використання .map) [cite: 354]
\`\`\`jsx
{postsData.map((post) => (
<Post
key={post.id}
author={post.author}
content={post.content}
date={post.date}
avatar={post.avatar}
/>
))}
\`\`\`

## [cite_start]Що станеться, якщо видалити атрибут key? [cite: 355]
Якщо видалити атрибут `key`, у консолі розробника з'явиться попередження: `Warning: Each child in a list should have a unique "key" prop`. React втратить здатність ефективно відрізняти елементи один від одного при алгоритмі узгодження (Reconciliation). Це призведе до падіння продуктивності при рендерингу, а також може викликати серйозні баги з відображенням даних, якщо список буде змінюватися (наприклад, при видаленні або сортуванні постів).

## Відповіді на контрольні запитання
1. **Навіщо потрібен проп key у списках React?**
   [cite_start]Відповідь пов'язана з ідентифікацією елементів при алгоритмі узгодження (Reconciliation)[cite: 344]. `key` допомагає React зрозуміти, які саме елементи змінилися, були додані або видалені, щоб оновити лише їх, а не перемальовувати весь список.
2. **Чому не рекомендується використовувати індекс масиву (index) як ключ?**
   [cite_start]Це може призвести до помилок при зміні порядку елементів або їх видаленні[cite: 345]. Якщо видалити перший елемент, всі індекси змістяться, і React може прив'язати старий стан до нових елементів.
3. **У чому різниця між імперативним створенням списку (цикл for) та декларативним (map)?**
   Імперативний підхід вимагає покрокового опису дій (створити масив, пройтися циклом, додати елементи). [cite_start]Декларативний підхід (через `.map()`) фокусується на тому, *що* ми хочемо отримати[cite: 346], описуючи трансформацію даних у компоненти, що робить код коротшим і передбачуваним.
4. **Як передати всі властивості об'єкта в компонент одним рядком?**
   [cite_start]Використовуючи Spread attributes: `{...item}`[cite: 347].
